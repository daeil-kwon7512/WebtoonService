# Django Webtoon Project

## 📅 이번주 작업 요약

- **기본 Django 틀 설정**
  - `accounts` (회원가입/로그인), `toons` (웹툰 CRUD/출력) 앱 분리 및 구조화
  - CustomUser 모델 도입 및 AUTH_USER_MODEL 세팅

- **외부 Webtoon API 연동/동기화**
  - 네이버, 카카오, 카카오페이지 플랫폼별 웹툰 1~50페이지 자동 동기화
  - 최초 접속 시 DB 비어있으면 전체 데이터 서버에 저장, 이후부터는 DB에서 빠르게 출력

- **웹툰 목록 조회 & 플랫폼별 필터링**
  - 상단 네비게이션에서 플랫폼 선택 시 각 웹툰 목록 동적 출력
  - 카카오/카카오페이지 플랫폼은 `is_end=False`(연재중) 웹툰만, 네이버는 전체 웹툰 출력

- **업데이트 요일 기준/검색 기능**
  - 업데이트 요일이 있는 웹툰만 출력
  - 작가명/제목 기반 검색창 구현

- **Json fixture export**
  - DB 내 웹툰 데이터를 JSON fixture로 추출해서 관리 가능하도록 구현

## 💡 이번주 작업에서 난이도 있었던/신경 쓴 점

- **API 페이징 수집 및 break 조건**  
  1~50page까지 돌며 빈 페이지에서 자동 종료(과한 반복 최소화)  
  API 구조 잡는 것이 실제 API와 맞아떨어져야 해서 신경 썼음

- **업데이트 요일/완결 필터/검색의 정확한 구현**  
  불러오는 웹툰 볼륨이 크기 때문에 완결 웹툰은 배제하고 싶은데 
  is_end 항목이 제대로 돼있는 웹툰이 잘 없음
  이 부분을 어떻게 처리할지 고민해야 할 듯

- **템플릿에서 authors 출력 방식**  
  파이썬-템플릿 데이터 타입(리스트/문자열) 혼동으로 출력 이상 오류  
  실제 저장 포맷과 출력단 로직 꼼꼼히 분리

## ✨ 앞으로의 개선/계획

- **Front-end 고도화**
  - 기본 Django 템플릿 → 추후 Vue 등 SPA 프론트엔드로 전환 예정
  - UX/UI 개선, 검색 고도화, 리스트/상세 페이지 분리

- **Back-end 구조화/고도화**
  - API 동기화, 에러/로깅 관리, 서비스 추상화
  - 웹툰 데이터 마이그레이션, 효율적 캐싱, 배치 동기화 개선

- **추천 시스템 구현**
  - 처음 회원가입하면 선호하는 웹툰 선택
  - 웹툰 기반으로 상단에 추천하는 웹툰 리스트업
  - 사용자의 웹툰 열람 기록을 바탕으로 추천 리스트업 업데이트 등

- **운영/배포**
  - GitHub 기반 코드관리 및 협업, Render 등 배포환경 자동화 준비
